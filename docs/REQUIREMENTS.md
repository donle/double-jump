## 1. 项目概述

### 1.1 一句话描述
一款 **2D 横版双人在线协作跳跃游戏**，两名玩家被一根弹性绳拴在一起，必须互相配合跨越越来越危险的地形。

### 1.2 项目目标
- 做一个**轻量、易上手、强调协作**的双人联机休闲竞技游戏。
- 核心体验：**一根绳子 + 重力 + 信任**。
- 目标平台：Web（浏览器），后续可拓展到 PC / 移动端。

### 1.3 核心卖点
- 强协作：单人掉下去不会死，但需要队友救。
- 强物理反馈：弹性绳索的真实拉扯感。
- 强可重玩性：地形程序化生成，距离越远难度越高。

## 2. 游戏类型与风格

| 维度 | 描述 |
|---|---|
| 类型 | 2D 横版动作 / 协作跳跃 / 平台跳跃（Platformer） |
| 视角 | 侧视 2D |
| 美术风格 | 简洁扁平风 / 卡通色块 / 留白多 / 强调角色和绳索辨识度 |
| 玩家人数 | 2 人（在线联机） |
| 单局时长 | 3–8 分钟 |
| 死亡规则 | 软死亡（单人）+ 硬死亡（双人） |

## 3. 核心玩法

### 3.1 基础规则
- 两名玩家从屏幕同一侧出发，向右前进。
- 两人之间始终被**一根弹性绳**连接。
- 通过跳跃、移动、协作使两人不断前进。
- 走得更远 → 关卡自动变难。

### 3.2 死亡与救援机制
| 情况 | 结果 |
|---|---|
| 一名玩家掉进坑里，另一名在安全地面 | 坑里的玩家不死，处于"悬挂"状态，由队友通过绳索拉住 |
| 队友通过拉绳 / 跳跃把悬挂者拽回地面 | 继续游戏 |
| 两名玩家同时掉进同一坑（或相互拉扯导致都掉下去） | 整局失败，回到起点 |
| 玩家掉出地图边界外 | 等同于掉坑 |

### 3.3 节奏
- 关卡为**无尽模式**或**按距离划分章节**。
- 每跳一段距离会有"难度梯度"提升。
- 单局结束条件：双人同时失败 / 主动退出 / 达到指定终点（可选）。

---

## 4. 角色与操作

### 4.1 角色设定
- 两个外观不同、颜色不同的角色（便于辨识，例如蓝 / 红）。
- 角色体积小，绳索粗，颜色清晰，对比度强。

### 4.2 操作（建议方案，最终由 AI 选型时确认）

| 按键 | 动作 | 说明 |
|---|---|---|
| `A / D` 或 `← / →` | 左右移动 | 地面与空中都可移动 |
| `Space`（长按） | 蓄力跳跃 | 蓄力越久跳得越远/越高，但有上限 |
| `Shift` | 拉绳 / 收绳 | 缩短两人距离 |
| `Q / E` | 抛绳 / 放绳 | 调整绳索张力 |
| `R` | 复活（被挂时由队友手动触发） | 队友把悬挂者拉起 |

> 双人需各自控制自己角色（**一人一套输入**），不共享键位。

## 5. 物理与绳索机制（核心）

### 5.1 绳索属性
- 长度可变、有**最大自然长度**与**最小自然长度**。
- 默认处于**自然长度**附近，呈轻微下垂。
- 超过自然长度 → 产生**弹力**，类似橡皮筋。
- 玩家可主动收绳 / 放绳。

### 5.2 受力规则
- 玩家受**重力**影响，跳跃高度有上限。
- 玩家 A 跳跃时，绳索会通过张力带动玩家 B。
- 玩家 B 在空中时由于绳索牵引，可被"甩"出去。
- 两人距离过近时，绳索会施加**排斥力**避免重叠。

### 5.3 关键参数（建议初始值，AI 可调）
- 重力：`g = 9.8`（可调）
- 跳跃初速度上限：例如 `15 m/s`
- 绳索自然长度：例如 `3 m`
- 绳索最大长度：例如 `6 m`
- 弹力系数 `k`：决定绳子硬度
- 阻尼系数 `d`：决定摆动衰减
- 移动加速度 / 摩擦力：分别设定

### 5.4 物理表现要求
- 绳子必须有**可见的形变曲线**（贝塞尔曲线 / 弹簧模拟）。
- 摆动、甩人、反弹等动作必须有**手感反馈**（轻微屏幕震动、音效）。
- 物理步长固定（建议 60 FPS 物理更新）。

## 6. 地形与关卡元素

### 6.1 地形类型（共 5 类）

#### 类型 ① 普通地面
- 不同高度的平台。
- 间距、宽度、垂直落差可调。

#### 类型 ② 坑（Hole / Pit）
- 玩家踩空即进入"悬挂"或"死亡"状态。
- 宽度可变。

#### 类型 ③ 浮空固定板
- 悬浮在空中、不移动。
- 玩家踩上去会停留。
- 间距较远，必须借助跳跃 + 绳索甩动才能通过。

#### 类型 ④ 浮空移动板
- 在指定路径上**水平 / 垂直 / 圆周**移动。
- 玩家需要在板移动到合适位置时起跳。
- 速度随距离增加而提高。

#### 类型 ⑤ 浮空限时板
- 出现后只能停留 **N 秒**（例如 1.5s）。
- 倒计时归零后板消失，玩家掉下去。
- 距离越远，停留时间越短。

### 6.2 关卡设计原则
- **程序化生成** + 难度系数 `d = distance / 1000`。
- `d` 越大：
  - 坑越多、越宽。
  - 浮空板间距越大。
  - 移动板速度越快。
  - 限时板停留时间越短。
  - 地形高度差越大。
  - 障碍组合更复杂（连续限时板 + 移动板等）。

## 7. 多人联机

### 7.1 联机要求
- 实时双人在线。
- 一台机器两人玩（分屏同屏）或两人远程联机（首选远程联机）。
- 低延迟（目标 < 100ms）。

### 7.2 通信架构
- **客户端-服务器架构**（C/S）。
- 服务器负责权威物理计算与状态同步。
- 客户端预测 + 插值，弱网也能流畅。

### 7.3 必须实现
- 房间系统（创建 / 加入 / 退出）。
- 准备 / 开始 / 失败 / 胜利状态同步。
- 断线重连（30s 内）。
- 简单的匹配（随机匹配 / 房间号匹配）。

### 7.4 数据同步
- 玩家位置、速度、绳索张力。
- 关卡种子（用于双方地形一致）。
- 双方输入帧（可选）。

## 8. UI / UX

### 8.1 界面
- 主页：开始游戏 / 创建房间 / 加入房间 / 设置。
- 房间页：玩家头像 / 准备按钮 / 房主开始。
- 游戏中：双角色同屏 + 绳索可视化 + 距离提示。
- 结束页：本局距离 / 历史最佳 / 再来一局。

### 8.2 视觉提示
- 绳索张力高时**变色或抖动**。
- 限时板倒计时**进度条 / 闪烁**。
- 队友悬挂时**提示箭头 / 拉绳按键高亮**。
- 玩家掉坑瞬间**屏幕红色 vignette**。

### 8.3 音效（基础即可）
- 跳跃、落地、绳子拉紧、掉坑、胜利 / 失败 BGM。
- 音量可调。

## 9. 技术栈建议（AI 可调整）

- **方案 A（推荐 Web）**：TypeScript + Phaser 3 / PixiJS + matter.js（物理）
- **方案 B**：Unity 2D（C#）
- **方案 C**：Godot（GDScript / C#）

- Node.js + TypeScript
- WebSocket（ws / socket.io）
- 自实现权威物理循环（可与客户端使用同一物理库）

### 9.3 协议
- Protobuf / JSON（建议 Protobuf，性能更好）

### 9.4 工具
- 包管理：pnpm / npm
- 构建：Vite / esbuild
- 测试：Vitest / Jest
- Lint：ESLint + Prettier

## 12. 验收标准（M1 Demo 通过条件）

1. 两个角色在浏览器中能跑、能跳、被绳子连着。
2. 玩家 A 跳起来时，玩家 B 会被绳子带动（看得见甩动效果）。
3. 一名玩家掉坑，**不会**判失败；悬挂状态下队友能把其拉回。
4. 两人同时掉坑，**判失败**，回到起点。
5. 5 种地形都至少出现一次，移动板和限时板动起来。
6. 距离越远，至少能感觉到**难度提升**（移动板更快、限时板更短）。
